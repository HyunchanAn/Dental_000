## 2026-07-14
- Dental_008 모듈(Instance Segmentation) RTX 5080 환경에서 재학습 완료
- 배치 사이즈 8, num_workers 8, AMP 적용으로 속도 대폭 개선 (10.9GB 데이터 약 1시간 소요)
- 학습 완료 후 Epoch 30 가중치(Loss: 0.3136)로 검증 수행
- 검증 결과: 추론 속도는 0.0636s/image(15.72 FPS)로 2배 향상되었으나, 정밀도(Precision)는 12.5% 수준에 머무름
- 높은 재현율(Recall 92%) 대비 과도한 배경 오탐지(False Positive 1144건)가 원인으로 확인됨
- 향후 해결 방안으로 후처리(NMS, Top-K Heuristic) 강화 및 RPN Negative Mining 파라미터 조정 검토 예정
- [긴급 추가 테스트] 후처리 필터(Score > 0.80, NMS IoU 0.15, Max 32 Boxes) 적용 결과: FP는 844건으로 여전히 높으나 재현율(Recall)이 92%에서 58%로 폭락함.
- 원인 분석: 모델이 정답인 실제 치아보다 오답인 턱뼈/배경에 더 높은 신뢰도(Confidence Score)를 부여하는 심각한 역전 현상이 확인됨.
- 결론: 후처리만으로는 성능 복구 불가 판정. 앵커박스 샘플링(OHEM) 파라미터 수정을 통한 단기 재학습 또는 2-Stage 디커플링 아키텍처로의 전환이 강제됨.
- [오버나이트 2-Stage 개편 및 테스트]
  - 1단계: 53개 클래스를 단일(Tooth) 클래스로 병합하여 30에포크 재학습 완료.
  - 2단계: 치열궁(U자형) 곡선 기반 동적 템플릿 매칭 휴리스틱 알고리즘(`arch_sequence_matcher.py`) 추가 완료.
  - 원인 및 향후 과제: 모델 구조상 Mask R-CNN의 고질적인 RPN 오탐지(턱뼈를 치아로 0.8 이상 확신)가 2-Class 변경 후에도 근본적으로 해결되지 않음. 향후 YOLOv8-seg 등 구조적으로 가볍고 배경-전경 분리에 특화된 1단계 전담 모델 교체가 필수적임.
- [YOLOv8-seg 전면 마이그레이션 적용 완료]
  - COCO 데이터셋을 YOLO 포맷으로 변환 후 단일 클래스(0: Tooth)로 YOLOv8m-seg 모델 30에포크 학습 진행 완료 (약 9분 소요).
  - 2단계 매칭 알고리즘과 통합된 평가 결과, 추론 속도가 18 FPS에서 46 FPS로 2.5배 폭증하였고, 치아 탐지 재현율(Recall)이 3배 상승(14.3% -> 42.8%)함.
  - 향후 성능 완성을 위해서는 배경 전용 네거티브 데이터셋(Hard Negative) 주입을 통한 모델의 오탐지 억제 훈련이 필요함.
- [자체 Hard Negative Crop 파인튜닝]
  - 기존 학습 이미지에서 치아와 겹치지 않는 턱뼈/빈 공간을 400x400 크기로 잘라내어 빈 라벨 파일과 함께 500장 생성 및 병합.
  - 기존 best 가중치를 이어받아 30 에포크 파인튜닝 진행.
  - 벤치마크 결과: FP(오탐지) 수치가 1174에서 1180으로 오히려 소폭 상승. 빈 라벨(배경) 데이터를 모델이 유의미한 네거티브 피처로 충분히 학습하지 못했거나, 파인튜닝 에포크/학습률 조정이 추가로 필요함. 외부 무치악 데이터 확보 등 보다 근본적인 배경 데이터 투입이 최종 해결책으로 보임.

### [Model Version History & Limitations]
1. **Model v1 (Mask R-CNN)**: 
   - 경위: 초기 DENTEX 평가용. 
   - 한계: 2-stage 구조라 속도가 느림(18 FPS), mAP가 낮고 Recall이 낮음(14%).
2. **Model v2 (YOLOv8m-seg, `yolov8m_seg_run`)**: 
   - 경위: 실시간성 및 성능 확보를 위해 1-Stage 모델로 마이그레이션. 
   - 한계: 속도(46 FPS)와 Recall(42%)은 비약적으로 상승했으나, 턱뼈/임플란트를 치아로 착각하는 오탐지(FP: 1174)가 극심함.
3. **Model v3-Crop (YOLOv8m-seg, `yolov8m_seg_finetune`)**:
   - 경위: 외부 데이터 부재로 기존 이미지의 빈 공간을 크롭해 500장 배경 데이터로 투입 후 파인튜닝.
   - 한계: 부분적인 패치 이미지만으로는 턱뼈의 해부학적 맥락을 학습하지 못해 FP 감소에 실패함(FP: 1180).
4. **Model v4-HardNegative (진행 예정)**:
   - 경위: 사용자가 직접 수집한 75장의 실제 무치악/임플란트 픽스처 엑스레이를 배경 데이터(빈 라벨)로 병합하여 '처음부터 다시 학습(Train from scratch)' 진행. 기존 가중치의 편향을 초기화하고 강건한 네거티브 특징을 학습할 예정.
