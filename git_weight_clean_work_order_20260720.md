## 1. 현재 상황 및 위협 분석 (Audit Results)
메인 워크스테이션에서 전체 서브모듈(001~014 및 Panoramic_Reader)을 대상으로 Git 트래킹 상태를 정밀 감사(Audit)한 결과, 다음과 같은 치명적인 스토리지 비대화(Bloat) 위협이 적발되었습니다. 

우리는 이미 100% HuggingFace 원격 캐싱(ONNX) 아키텍처로 전환을 완료했으므로 소스코드 저장소에 가중치 파일이 존재할 이유가 전혀 없음에도 불구하고, 과거의 잔재들이 Git 인덱스에 남아있습니다.

* **일반 Git 강제 트래킹 (최악의 상태)**: LFS조차 거치지 않고 순수 Git 히스토리에 무거운 가중치가 추적되어 Clone 속도를 저하시키는 모듈들
  - `Dental_002` (models/best_refined.pt)
  - `Dental_008` (weights/yolov8m-seg.pt)
  - `Dental_011` (src/weights/*.pth)
  - `Dental_012` (models/best.onnx)
  - `Dental_Panoramic_Reader` (runs 및 weights 디렉토리 내 다수의 .pt 파일)

* **불필요한 LFS 트래킹 방치**: 허깅페이스 전환으로 인해 더 이상 관리할 필요가 없는 폐기 대상 가중치들이 LFS 대역폭을 낭비 중인 모듈들
  - `Dental_001`, `Dental_004` (수많은 에포크 스냅샷), `Dental_005` 등

(.gitignore에 명시되어 있더라도 과거에 `git add` 된 이후 `git rm --cached` 처리가 되지 않아 발생한 현상입니다.)

## 2. 조치 지시 (Action Items)

파이프라인 랩탑은 본 지시서를 확인하는 즉시 다음 절차에 따라 전체 리포지토리 다이어트 작업을 수행하십시오.

1. **Git 인덱스 적출 (`git rm --cached`)**
   - 모든 서브모듈(Dental_000 루트에서 `git submodule foreach` 활용 권장)을 순회하며 다음 명령어를 실행하여 Git 인덱스에서 가중치를 완전히 도려내십시오.
   - `git rm -r --cached "*.pt" "*.pth" "*.onnx"` (에러 무시 옵션 포함)

2. **`.gitignore` 방어벽 재점검**
   - 위 명령어로 적출된 파일들이 다음 커밋에 다시 딸려 들어오지 않도록, 각 모듈의 `.gitignore` 파일에 `*.pt`, `*.pth`, `*.onnx` 규칙이 정상 작동하는지 교차 검증하십시오.

3. **로컬 물리 파일 삭제 (선택/권장 사항)**
   - 이미 모든 서비스가 HuggingFace에서 `hf_hub_download`를 통해 ONNX를 로드하므로, 로컬 디스크 공간 확보를 위해 해당 찌꺼기 파일들을 물리적으로 삭제(rm)하는 것을 강력히 권장합니다.

4. **전역 커밋 및 푸시 (Global Sync)**
   - 적출 작업이 완료된 서브모듈들을 각각 Commit & Push 하신 뒤, 루트 모듈(`Dental_000`)의 서브모듈 포인터 또한 업데이트하여 Push 하십시오.

# (파이프라인 랩탑) 가중치 제거 및 다이어트 작업 계획 (Implementation Plan)

메인 워크스테이션에서 지적하신 Git 스토리지 비대화 문제를 완벽히 해결하기 위해 다음과 같이 일괄 다이어트 및 클린업 작업을 진행할 계획입니다.

## 1. 전역 Git 인덱스 가중치 추적 해제
루트 디렉토리(`Dental_000`) 및 모든 하위 서브모듈(Dental_001~014, Panoramic_Reader)을 대상으로 순회 스크립트를 작성하여 아래 명령을 일괄 실행하겠습니다.
- `git rm -r --cached "*.pt" "*.pth" "*.onnx"` 

## 2. `.gitignore` 전수 조사 및 패치
각 모듈의 `.gitignore` 파일에 다음 항목이 누락되어 있다면 자동 추가하여 재발을 방지하겠습니다.
```gitignore
# Model Weights
*.pt
*.pth
*.onnx
```

## 3. 물리적 가중치 파일 일괄 삭제
현재 모든 모듈이 `app.py`에서 `hf_hub_download`를 통해 Lazy Loading으로 ONNX를 주입받고 있으므로, 로컬 공간만 차지하고 있는 모든 `.pt`, `.pth`, `.onnx` 더미 파일들을 쉘 스크립트(`Remove-Item` 등)를 통해 **물리적으로 완전히 삭제**하겠습니다.

## 4. 전역 커밋 및 원격 저장소 동기화 (Sync)
- 모든 서브모듈에 대해 `git add .`, `git commit -m "chore: remove tracked weights and update gitignore"`, `git push`를 병렬 또는 순차적으로 실행.
- 루트 리포지토리(`Dental_000`)에서 갱신된 서브모듈 포인터들을 커밋하고 Push하여 메인 워크스테이션에서도 즉시 갱신이 가능하도록 조치.
